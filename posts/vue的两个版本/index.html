Vue的两个版本

2020年3月3日

@vue/cli的安装与使用

在选择vue版本之前，需要有一个vue项目，这里就需要使用@vue/cli

    npm install -g @vue/cli
    //or
    yarn global add @vue/cli

    //安装好后创建项目
    vue create 项目名

    //根据需求选择配置

    //yarn serve  /  webpack-dev-server  /npm run serve   预览

vue两个版本区别

  • 完整版的引入方式 :
    1. cdn引入 vue.js/vue.min.js
    2. webpack / @vue/cli 引入 ： 查看文档，确保正确

完整版可以从HTML或template直接得到视图,有compiler（编译器）可以将含有占位符{{}}或者条件语句变成真实的DOM节点，compiler会占用一定的代码体积，比不完整版大40%

  • 完整版的引入方式 :
    1. cdn引入 vue.runtime.js/vue.runtime.min.js
    2. webpack / @vue/cli 引入 ： 查看文档，确保正确

没有compiler,不能得到HTML,template的视图。需要自己写h函数，也可以将html写在vue文件中 在通过vue-loader将vue文件中的html转换为h函数 （h函数就是createElement的缩写）

template怎么用

template只有在完整版中有用

//vue文件中的template标签
  <template>
      <div id="app">      
          {{n}}
          <button @click="add">+1</button>   
     </div> 
  </template>

//js中的template
    template : `
        <div id="app">      
          {{n}}
          <button @click="add">+1</button>   
        </div> 
    `

render的使用方法

//不完整版在js中构建视图
  render(h){ 
       return h('div', [this.n,h（'{on:{click:this.add}’,'+1'])
   }

//不完整版使用vue-loader

//先创建一个demo.vue文件，在里面构建视图
    import demo from "./demo.vue"
     new Vue({
       el: "#app",
       render(h) {
         return h(demo)
       }
     })

使用codesandbox.io

这是一个网页版的vscode 传送门

建议：不要注册，直接用。不然项目会有数量限制

  1. 进入网站后选择创建项目（create sandbox）
  2. 选择vue
  3. 一个vue项目就创建完成了！！！

如果要保存这个项目

点击file里的export to ZIP 就会下载这个项目的压缩包 只要解压缩就能在本地打开了。

这是一个非常适合动手学习的网站，还有一些功能大家可以自己深入了解一下。
